# 08 — Solutions (by role) + Trust/Security + Customers

> Secondary pages that convert specific buyers and de-risk enterprise. Build after Home + product pillars.

## A. Solutions / by-role pages (`/solutions/*`)
Same engine, different framing per buyer:
1. **AI Engineers** (`/solutions/ai-engineers`) — "Find the failures your evals miss; prove fixes fast." Dev-centric, SDK, replay, CI.
2. **Engineering Leaders** (`/solutions/eng-leaders`) — "Ship agents to production with confidence. Prove reliability, prevent regressions, quantify failure cost." ROI + blast-radius $ framing.
3. **Platform / SRE** (`/solutions/platform`) — "A reliability gate for every agent team. Self-host, RLS, audit, SSO." Governance + scale.

Each: tailored hero + 3 pillar reframes + role-specific proof + CTA. (Optional v2: by use-case — refund/support/sales agents.)

## B. Trust / Security (`/security` or `/trust`)
> Enterprise blocker-remover. Critical for the ICP (production agents).
- **Capture is fail-closed** — never breaks your production call path.
- **Data handling** — PII masking, retention controls, provider-key vault (AES-GCM envelope), RLS tenant isolation.
- **Deployment** — cloud / self-host / VPC; private replay worker (enterprise).
- **Access** — SSO/SAML, audit logs (enterprise).
- **Honesty posture** — "we never fabricate a verified result."
- security.txt, responsible disclosure contact, subprocessors list.

## C. Customers / Case studies (`/customers`)
- **Until real customers:** keep minimal — OSS adoption stats, "built for production agents," a single design-partner story when available.
- **The killer case study** (when ready): one real agent → discovered unknown failure → verified fix → blocked regression. This one story sells more than any feature page. (Tied to completion plan Phase 7.)
- Don't fake logos or quotes — credibility is the whole brand.

## My POV
- Role pages matter for sales but **don't build them before the product is proven** — they're amplifiers, not the core.
- The **Trust/Security page is worth doing early** for the enterprise ICP — fail-closed + data handling + honesty removes the biggest objections.
- Resist fake social proof. An empty-but-honest customers page beats fabricated logos for a dev-trust brand.
